# Le Rouge — правила работы

## ⚠️ ГЛАВНОЕ ПРАВИЛО: детальный анализ дизайна ПЕРЕД версткой

Перед тем как верстать любую секцию — **обязательно делать детальный разбор Figma через `get_design_context`**, а не только скриншот. Выписывать ВСЕ элементы:
- фоновые изображения и декоративные слои (волны, свечения, glow, sticky-фоны)
- разделительные линии между блоками
- точные размеры/позиции/наезды (margin-overlap, translate)
- все тексты и их типографику (mixin + размер)
- annotation-атрибуты (`data-annotations`) — там комментарии заказчика (параллакс, sticky, плавная смена фона и т.д.)

Причина: при работе «по скриншоту» уже было упущено много элементов (фоны-волны, свечения, GOLD/SILVER карты, sticky-фон девушки, разделительные линии). Каждый раз приходилось переделывать.

**Чеклист перед версткой секции — ОБЯЗАТЕЛЕН, без исключений:**
1. `get_design_context` ноды секции — прочитать ВСЮ структуру
2. `get_metadata` ноды — пройтись по ВСЕМ дочерним фреймам, ничего не пропустить
3. **Выписать ПОИМЁННО ВСЕ child-фреймы корневого узла** (особенно искать имена: `свет`, `glow`, `фон`, `bg`, `wave`, `волна`, `линия`, `line`, `divider`, `градиент`, `gradient`, `тень`, `shadow`). Если есть — значит обязательны к реализации.
4. Найти `data-annotations` — это требования заказчика (параллакс, sticky и т.д.)
5. Проверить какие картинки есть (Figma часто отдаёт пустой PNG для bitmap → брать через `get_screenshot` ноды)
6. **Перед тем как писать код — отписать пользователю списком: «нашёл X фреймов, реализую: ...»** — чтобы не пропустить ничего
7. Только потом верстать

❌ Типичные пропуски (история):
- декоративный «свет» с эллипсами по краям
- разделительные линии между блоками
- sticky-фоны (волна, фото-bg)
- 4 GOLD/SILVER карты в loyalty
- glow за хедером в show-video (Group71)
- top/bottom градиенты затемнения секций

## ⚠️ ПРАВИЛО: детальный разбор кнопки/элемента (не только секции)

Когда берёшь из Figma **кнопку или отдельный элемент** — так же детально разбирай ИМЕННО его внутренности, по каждому свойству, чтобы не было неточностей. Разобрать поимённо: заливку, **обводку (stroke)**, тень, blur, иконку, текст, радиусы, паддинги, **все состояния** (покой/ховер/актив).

**Важно — `get_design_context` упрощает и теряет детали:**
- **Градиентную обводку отдаёт как ОДИН сплошной цвет** (например `border-[#fff5e5]`), хотя в дизайне это градиент. ВСЕГДА сверять обводку со скриншотом/визуалом: если рим переливается (яркий с одной стороны → мягкий с другой) — это градиентный stroke, делать через `border: solid transparent` + `background: <fill> padding-box, <gradient> border-box`.
- **Тени (drop-shadow) часто не попадают** в код — брать со `get_screenshot` ноды.
- **Полупрозрачные fill** (alpha 0.9→0.2 и т.п.) на плоском фоне кадра Figma выглядят сочно, но в проекте блендятся с фоном за элементом (фото hero, белый заголовок просвечивает) → выглядят мутно. Если нужно «как в кадре Figma» — подложить непрозрачную тёмную базу (`linear-gradient(#2b2b2b,#2b2b2b) padding-box`). Сплошной цвет в `background` допустим ТОЛЬКО в последнем слое.
- Брать ОБА состояния ноды (покой/ховер) и сверять, что меняется: цвет, бордер, фон, blur, иконка.

Примеры готовых эталонов: `src/features/go-btn/go-btn.scss` (круглая GO — градиентный бордер + тёмная база + тень), `src/shared/ui/button/button.scss` → `.ghost` (МЕНЮ — градиентный бордер яркий-слева→мягкий-справа).

## Стек / архитектура
- Pug + SCSS + Vanilla JS, сборка Webpack, FSD-архитектура
- Брейкпоинты: `$desktop` (1400), `$tab-sm` (768). Дизайн-сетки Figma: 1920 / 1440 / 360
- Все размеры через `rem()`, цвета через CSS-переменные (`--gold-color #F1B965`, `--beige-color #FFF5E5`, `--black-color #0B0B0B`)
- Типографика — миксины из `_mixins.scss` (display-l/m/s, h1-h6, body-xl/l/m/s/xs, descr-md/sd/s/xs)
- Подключение новой секции: `mixins.pug` (include) + `main.scss` (@use) + `home.pug` (+mixin) + `app/index.js` (если есть JS)

## Картинки из Figma
- `get_design_context` возвращает пустой/битый PNG для растровых фото → брать через `get_screenshot` нужной ноды
- Раскладывать в `src/shared/img/<section>/`
- Сжимать крупные через `sips` или `python3 PIL`

## Прод-сборка
- `npm run build` после каждой секции для проверки
- Билд ~2 минуты (картинки минифицируются), это норма
