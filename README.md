# Paradigma Digital Frontend template

основа для архитектуры приложения взяты с https://feature-sliced.github.io/documentation/ru/docs/get-started/overview

## ---папки в проекте---

# APP

это папка с основными настройками для приложения

templates - тут лежат templates для Pug, и миксины для него же

styles - тут лежат глобальные стили проекта - миксимы, переменные, функции, шрифты и т.д.
там же лежит основной файл для стилей main.scss куда мы подключаем компоненты стилей

index.js - основной js файл, куда подключаются скрипты компонентов

# ENTITIES

эта папка для сущностей, например условный user, product-card и т.п.

# FEATURES

папка для компонентов которые предоставляют какую-либо бизнесс ценность - корзина, поиск, форма, фильтры и т.п.

# WIDGETS

папка для цельных, самодостаточных блоков - products-swiper, hero, header, footer и т.п.

# PAGES

папка для страниц

создать страницу чтобы она попала в папку dist можно двумя способами

src/pages/index.pug - тут мы просто создаем в корневой папке pages файл со страницей, на выводе у нас в dist получится "dist/index.html"

src/pages/home/home.pug - тут мы создает дочернюю папку для страницы, и уже в ней создаем файл страницы, на выводе в dist мы получим "dist/home.html"

второй вариант будет использоваться реже, он нужен для того чтобы туда можно было положить компоненты которые "эксклюзивны" только для конкретной страницы

такие компоненты будут лежать в папке ui рядом с самой страницей, эти компоненты не будут генерироваться как отдельные страницы
src/pages/home/ui/some-component/some-component.pug

# SHARED

какой либо переиспользуемый код/переменные/компоненты

например в этой папке будут лежать все ui компоненты для сайта - кнопки, селекторы, инпуты и т.д.
так же в ней лежат все статические файлы для сайла - шрифты, какие-либо файлы, спрайты, изображения
так же в ней лежат helpers js функции которые могут переиспользоваться по всему сайту, для таких функций отведена папка lib

в нашей реализации так же в этой папке в ui будут лежать модальные окна
найти их можно по пути src/shared/ui/modals/

это не лучшее решение для модальных окон, но другое место для них я придумать пока не смог
